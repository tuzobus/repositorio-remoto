Sobre la salsa

Preguntas del laboratorio 1

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • El internet es una red de computadoras que abarca todo el planeta, las cuales están conectadas entre sí, mientras que WWW es la colección de páginas que se alojan en esta red.

  • ¿Cuáles son las partes de un URL?

    • Un URL empieza con el esquema (http o https), el nombre de host (www.ejemplo.com), la ruta (/ejemplo/index.html), y una cadena de consulta (proporciona parámetros para que un recurso pueda usarlos, como en una búsqueda)

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • GET recupera datos de un recurso específico. HEAD pide una respuesta idéntica a GET pero sin el cuerpo de la respuesta. POST envía una entidad a un recurso específico. PUT reemplaza las representaciones actuales del recurso de destino con el contenido de la petición. PATCH aplica modificaciones parciales a un recurso. DELETE borra un recurso específico.

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • Se debe utilizar el método POST pues este permite enviar varios datos en el cuerpo de la solicitud, además de que estos datos no se incluyen en la URL y se evita que queden registrados en el historial.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • El método que se utiliza es GET, pues permite que los parámetros puedan ir en la URL efectivamente.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • Este código 200 significa que la solicitud se procesó correctamente y no pasó ningun error.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • Lo más probable es que sí, pued un error 404 significa que un archivo no se encontró en el servidor, por lo que pudo haber sido eliminado por el mismo desarrollador o la configuración de rutas esté incorrecta.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • Un error 500 significa que el servidor falló al intentar ejecutar una solicitud, por lo que existe la probabilidad de que el error radique en el servidor, lo cual ya no le correspondería a un desarrollador web.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    • Un atributo despreciado básicamente significa que está obsoleto. Ejemplos de esto pueden ser: font, center, big, small y applet (para js)

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • HTML 5 introduce mejoras respecto al 4. El 5 introduce etiquetas semánticas como header y footer, nuevos tipos de inputs, tiene una mejor integración con CSS, y mejora su renderización.

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    • Una tabla en HTML está conformada por varias etiquetas. table es el contenedor principal de la tabla. tr es cada línea horizontal dentro de la tabla. th son las celdas de encabezado. td contiene información dentro de la tabla. caption contiene el título o la descripción de la tabla

  • ¿Cuáles son los principales controles de una forma HTML5?

    • HTML tiene tipos de controles de entrada para formularios, como input, el cual a su vez tiene varios tipos que facilitan la organización y distinguen los tipos de datos de entrada. Algunos ejemplos de estos tipos de input son: text, password, email, number, url, etc.

  • ¿Qué tanto soporte HTML5 tiene el navegador que utilizas?

    • Mi navegador, Arc, tiene un puntaje de 525/555 puntos, por lo que tiene un soporte relativamente bueno de HTML5.

  • ¿Cuál es el ciclo de vida de los sistemas de información?

    • Planificación->análisis de requerimientos->diseño del sistema->implementación->pruebas->instalación->uso

  • ¿Cuál es el ciclo de desarrollo de sistemas de información?

    • Identificación del problema->planificación->definición de los requisitos->diseño y creación de prototipos->desarrollo de software->evaluación->despliegue->operaciones y mantenimiento

Preguntas del laboratorio 3

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    • !important es una regla que supera en prioridad a cualquier otra especificación en un archivo css, por lo que el diseño de cascada queda básicamente ignorado al usarse !important en alguna propiedad. Como desarrollador de software, recomendaría evitarlo porque el orden creado a partir de las especificaciones que van desde los selectores por elemento que cubren todo el documento hasta los más específicos por clase, se ve básicamente ignorado y roto por el uso del mismo.

  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    • Hay varias razones por las cuales debería escogerse con cuidado, la más evidente serían los derechos de autor y cómo elegir una imagen con copyright podría traer problemas legales. En un sentido orientado a diseño, una imagen mal elegida podría brindar mal contraste para el contenido de la página, o simplemente podría tener un mal aspecto al establecerla como fondo.

  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    • Estas unidades de estilo dependen del contexto en que se quieran utilizar. El porcentaje es mejor cuando se busca un diseño responsivo relativo al contenedor padre, los pixeles cuando se necesita un tamaño fijo, y los puntos cuando se buscan medidas físicas, pero no es bueno usarlo para pantallas, pues no interpretan estos tamaños físicos con su tamaño real.

    • Al usar una versión minimizada del CSS el tamaño del archivo disminuye, y por lo tanto tarda menos en cargarse en el navegador.

Preguntas del laboratorio 4

  • ¿Qué diferencias y semejanzas hay entre Java y JavaScript?

    • Las principales diferencias entre java y JS, que son lenguajes completamente diferentes, radica en que java requiere de la java virutal machine para compilares, mientras que JS puede compilarse en tiempo real en un navegador. Java se utiliza principalmente para desarrollar apps de escritorio, mientras que JS para desarrollar aplicaciones web.

  • ¿Qué métodos tiene el objeto Date? (Menciona al menos 5*)

    • Date.now() devuelve el número de milisegundos transcurridos desde el 1 de enero de 1970 a las 00:00:00 UTC. getMonth() devuelve el mes actual (empieza en 0 (enero)). getDay() devuelve el día de la semana (empieza en 0 (lunes)). getHour() devuelve la hora actual. getFullYear() devuelve el año como 4 dígitos (XXXX).

  • ¿Qué métodos tienen los arreglos? (Menciona al menos 5*)

    • push() agrega un elemento al final del arreglo. pop() elimina el último elemento del arreglo y lo devuelve. shift() elimina el primer elemento y lo devuelve. slice() devuelve una copia del arreglo desde un elemento inicial a uno final. splice() elimina elementos desde un elemento inicial a uno final

  • ¿Cómo se declara una variable con alcance local dentro de una función?

    • Para esto se utiliza la palabra "let".

  • ¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

    • Usar variables globales puede ser contraproducente en el sentido de que varias funciones no podrán usar el mismo nombre de variable con contenidos diferentes, aunque esto también podría verse como una ventaja al no tener que declarar la misma variable varias veces, todo depende del contexto.
